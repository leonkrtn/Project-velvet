Bekannte Bugs (dokumentiert)
Beide im CLAUDE.md als ⚠️ CRITICAL markiert
Bug 1 — Berechtigungen ohne Wirkung

Der Veranstalter konfiguriert Rechte für Dienstleister → UI schreibt in die neue Tabelle dienstleister_permissions.

Das Vendor-Portal liest Tab-Sichtbarkeit aber noch aus der alten Tabelle permissions.

Folge: Was der Veranstalter im Editor einstellt, hat keinen sichtbaren Effekt auf das Vendor-Portal.

Bug 2 — Middleware-Check für Veranstalter-Routes immer false

In middleware.ts:

if (isOrganizerRoute && isApproved === false && isApproved !== undefined)

Diese Bedingung ist immer false — ein Wert kann nicht gleichzeitig === false und !== undefined sein.

Folge: Jeder eingeloggte User könnte Veranstalter-Routen aufrufen (RLS in der DB schützt aber die Daten). Die eigentliche Zugriffskontrolle läuft über profiles.is_approved_organizer.

ℹ
Beide Bugs sind bekannte technische Schulden aus der Migration vom alten zum neuen System, nicht versehentlich eingebaut.

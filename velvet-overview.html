Bekannte Bugs (dokumentiert)
Beide sind im CLAUDE.md als bekannt markiert
Bug 1 — Berechtigungen ohne Wirkung

Der Veranstalter konfiguriert Rechte für Dienstleister in der UI. Die UI schreibt in die neue Tabelle dienstleister_permissions.

Aber: Das Vendor-Portal (VendorDashboardClient.tsx) liest Tab-Sichtbarkeit noch aus der alten Tabelle permissions.

Folge: Was der Veranstalter im Berechtigungs-Editor einstellt, hat keinen sichtbaren Effekt auf das was der Dienstleister im Portal sieht.

Bug 2 — Middleware-Check für Veranstalter-Routes immer false

In middleware.ts steht:

if (isOrganizerRoute && isApproved === false && isApproved !== undefined)

Diese Bedingung ist immer false, weil ein Wert nicht gleichzeitig === false UND !== undefined sein kann — false !== undefined ist zwar wahr, aber dann müsste der erste Teil auch wahr sein, was nur geht wenn isApproved genau false ist.

Folge: Jeder eingeloggte User könnte technisch Veranstalter-Routen aufrufen (sofern kein anderer Schutz greift). Die eigentliche Zugriffskontrolle läuft über profiles.is_approved_organizer in der DB via RLS.

ℹ
Beide Bugs sind im CLAUDE.md unter "⚠️ CRITICAL" dokumentiert und bekannt. Sie sind nicht durch Versehen entstanden, sondern stellen technische Schulden aus der Migration von alt zu neu dar.
